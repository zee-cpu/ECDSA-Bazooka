================================================================
CRITICAL FIXES APPLIED — ECDSA NONCE RECOVERY ENGINE
================================================================

PROBLEM
-------
- With 500 signatures and 12-bit MSB bias the tool was:
  1. Detecting the bias (good)
  2. Running lattice but producing wrong key
  3. "Verification" always passed (even for garbage d) because
     s' == s is an algebraic identity for any d when using w + x*d
  4. Real PubKey verification was a stub that always returned true

SOLUTIONS IMPLEMENTED
---------------------
1. Real secp256k1 point multiplication (secp256k1.cpp + .h)
2. Proper d*G == PubKey check in utils.cpp + verifier.cpp
3. PubKey is now the PRIMARY and authoritative verification
4. s-recompute check is now only for debug/telemetry
5. Hard rejection in recovery_engine when PubKey present but fails
6. Improved lattice basis (scaled 2^l * N form used in real papers)
7. Much better MSB bias estimator in bias_profiler (tries many d candidates)
8. Lattice solver now tries several leak values + scores candidates

FILES CHANGED
-------------
- src/secp256k1.cpp (new)
- include/secp256k1.h (new)
- src/utils.cpp (real compute_pubkey + verify_pubkey)
- src/verifier.cpp (PubKey is primary)
- src/recovery_engine.cpp (hard PubKey gate)
- src/lattice_solver.cpp (better basis + candidate search)
- src/bias_profiler.cpp (much better MSB detection)

HOW TO USE ON YOUR MACHINE
--------------------------

cd ~/Downloads/new/ecdsa_nonce_recovery

# 1. FULL CLEAN REBUILD (mandatory)
rm -rf build
mkdir -p build

cd build
cmake ..
make -j$(nproc)

# 2. Regenerate the exact failing test
cd ..
python3 scripts/generate_mock_signatures.py \
    --count 500 \
    --bias msb \
    --bias-bits 12 \
    --output data/4000.txt \
    --seed 424242

# 3. Run
cd build
./ecdsa_nonce_recovery -i ../data/4000.txt -v

# Expected good outcome:
# Either:
#   - Recovers 0xbf7add145323ea0861cb147b3e09d36515c83333a04b486fa63376f81227b50
#     and says "PubKey match: YES"
# Or:
#   - Reports failure with "PubKey match: NO" (if lattice still not strong enough)

IMPORTANT
---------
- 500 signatures for 12-bit bias is on the low side. Real papers often use
  800-2000+ for reliable recovery at ~12 bits.
- The profiler may still underestimate leaked bits — the lattice code now
  tries several values around the estimate.
- If it still fails with 500 sigs, try generating 1200-2000 signatures.

You now have strict verification + much better lattice code.
